Git Checklist for Beginners

  • 1. Install Git

    Download and install Git from the official Git website.

  • 2. Configure Git

    Set up your name and email for Git:

    git config --global user.name "Your Name"
    git config --global user.email "you@example.com"
  • 3. Initialize a Repository

    Create a new repository:

    git init

    This command initializes a new Git repository in your project folder.

  • 4. Check the Repository Status

    Check the current state of your repository:

    git status

    This shows which files have been modified or staged for commit.

  • 5. Add Files to Staging

    Stage your changes:

    git add .

    The . stages all files. You can specify individual files if needed.

  • 6. Commit Changes

    Save your changes to the repository:

    git commit -m "Initial commit"

    Always use descriptive commit messages.

  • 7. Connect to a Remote Repository

    Add a remote repository (e.g., on GitHub):

    git remote add origin https://github.com/username/repo.git

    Replace username/repo.git with your repository URL.

  • 8. Push Changes to Remote

    Push your changes to the remote repository:

    git push -u origin main

    The -u flag sets the upstream branch for future pushes.

  • 9. Pull Updates from Remote

    Sync your local repository with the remote:

    git pull origin main

    This fetches and merges changes from the remote repository.

  • 10. Create a Branch

    Create and switch to a new branch:

    git checkout -b feature-branch

    Use branches for new features or experiments.

  • 11. Merge a Branch

    Merge your branch back into the main branch:

    git checkout main
    git merge feature-branch
  • 12. Resolve Merge Conflicts

    If conflicts arise during a merge, resolve them in your editor, then:

    git add .
    git commit
  • 13. View Commit History

    See the commit history:

    git log
  • 14. Clone a Repository

    Clone an existing repository to your local machine:

    git clone https://github.com/username/repo.git
  • 15. Best Practices
    • Commit frequently with clear messages.
    • Use branches for features or fixes.
    • Pull changes before starting new work.
    • Keep your main branch stable.
